Bezpieczeństwo i jakość

Warstwa bezpieczeństwa

  • Sesja: podpisany cookie HMAC-SHA256 (Web Crypto, działa na Edge i w Node).
  • Proxy/middleware (proxy.ts): gating tras, role (admin/editor/viewer), redirect na /login.
  • CSP per-request z nonce (anty-XSS), frame-ancestors 'none'.
  • Anti-nuke / anti-raid / automod po stronie bota; webhooki weryfikowane HMAC (Stripe, Twitch, Ko-fi).
  • Atomowa ekonomia: RPC Postgres + per-user lock (brak wyścigów / dublowania salda).

Wdrożenie

Deploy i dane

Panel → Vercel

Next.js na Vercel; zmienne w Project Settings → Environment Variables; redeploy po zmianie.

Dane → Supabase

Schemat z dashboard/scripts/*.sql (strażnik schema:check pilnuje _ALL.sql).

Bot → host 24/7

Proces Node (np. Railway); token + dostęp do tej samej bazy.

Billing → Stripe

Env-gated; webhook /api/billing/webhook (HMAC) ustawia guilds.tier.
